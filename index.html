LG

LangGraph Overview

Low-level orchestration for stateful agents
Khái niệm Framework build agent/workflow bằng graph có state, control flow và persistence.
Khi dùng Agent nhiều bước, human-in-the-loop, long-running workflow, multi-agent.
Install pip install -U langgraph
Core APIs Graph API, Functional API, prebuilt agents, LangGraph Platform.
pip install -U langgraph langchain
pip install -U langchain-openai

# Main idea:
# StateGraph defines state.
# Nodes update state.
# Edges decide next node.
# Checkpointers persist execution.

Best: dùng LangGraph khi cần state/control flow rõ hơn chain hoặc agent đơn giản.

Mistake: build graph phức tạp khi create_agent hoặc LCEL chain đã đủ.

SG

StateGraph

Main graph builder for stateful workflows
Khái niệm Builder định nghĩa schema state, nodes, edges rồi compile thành runnable graph.
Khi dùng Workflow có shared state và nhiều bước điều phối.
Import from langgraph.graph import StateGraph, START, END
Methods add_node, add_edge, add_conditional_edges, compile
from typing import TypedDict
from langgraph.graph import StateGraph, START, END

class State(TypedDict):
    topic: str
    joke: str

def write_joke(state: State):
    return {"joke": f"Short joke about {state['topic']}"}

builder = StateGraph(State)
builder.add_node("write_joke", write_joke)
builder.add_edge(START, "write_joke")
builder.add_edge("write_joke", END)

graph = builder.compile()
result = graph.invoke({"topic": "graphs"})

Best: type state rõ bằng TypedDict hoặc Pydantic; node trả partial state.

Mistake: mutate state trực tiếp hoặc quên compile graph trước khi invoke.

ST

State & Reducers

Shared data and merge behavior
Khái niệm State là data graph đọc/ghi; reducer quyết định cách merge updates.
Khi dùng Append messages, aggregate results, parallel branches.
Import Annotated, add_messages, operator.add
Default Không reducer thì update overwrite key hiện tại.
from typing import Annotated, TypedDict
import operator
from langgraph.graph.message import add_messages

class State(TypedDict):
    messages: Annotated[list, add_messages]
    search_results: Annotated[list[str], operator.add]
    user_id: str

def add_result(state: State):
    return {"search_results": ["doc-1"]}

# messages append intelligently;
# search_results list concatenates;
# user_id overwrites if returned.

Best: đặt reducer cho keys được nhiều node/branch update song song.

Mistake: để list bị overwrite khi muốn append, hoặc reducer không deterministic.

ND

Nodes

Work units that read state and return updates
Khái niệm Node là function/runnable chạy một bước trong graph.
Khi dùng Model call, tool call, retrieval, validation, transform, routing prep.
Signature node(state, config=None, runtime=None)
Return Dict partial state, Command, hoặc interrupt flow.
from langchain.chat_models import init_chat_model

model = init_chat_model("openai:gpt-5.5")

def call_model(state: State):
    response = model.invoke(state["messages"])
    return {"messages": [response]}

def validate(state: State):
    if not state["messages"]:
        return {"errors": ["missing messages"]}
    return {}

builder.add_node("model", call_model)
builder.add_node("validate", validate)

Best: node nhỏ, idempotent khi có thể, output đúng schema state.

Mistake: để side effect nguy hiểm trong node mà không có interrupt/approval.

ED

Edges & Routing

Control flow between nodes
Khái niệm Edge quyết định node tiếp theo; conditional edge dùng routing function.
Khi dùng Branching, tool loop, approval flow, error path.
Constants START, END
Methods add_edge, add_conditional_edges
from typing import Literal
from langgraph.graph import START, END

def route(state: State) -> Literal["tools", "final"]:
    last = state["messages"][-1]
    return "tools" if last.tool_calls else "final"

builder.add_edge(START, "model")
builder.add_conditional_edges(
    "model",
    route,
    {"tools": "tools", "final": END},
)
builder.add_edge("tools", "model")

Best: route function trả label rõ, type hint bằng Literal để graph dễ hiểu.

Mistake: tạo vòng lặp không có điều kiện dừng hoặc route trả node không tồn tại.

CS

Command & Send

Update state and control routing dynamically
Command Node vừa update state vừa quyết định goto.
Send Fan-out động cho map-reduce hoặc parallel work.
Import from langgraph.types import Command, Send
Khi dùng Dynamic routing, parallel per-item processing, subgraph navigation.
from typing import Literal
from langgraph.types import Command, Send

def decide(state: State) -> Command[Literal["research", "answer"]]:
    if state.get("needs_research"):
        return Command(update={"status": "researching"}, goto="research")
    return Command(goto="answer")

def fan_out(state: State):
    return [
        Send("process_doc", {"doc": doc})
        for doc in state["docs"]
    ]

builder.add_conditional_edges("split", fan_out)

Best: dùng Command khi update và routing gắn chặt trong cùng quyết định.

Mistake: lạm dụng dynamic routing làm graph khó đọc hơn conditional edges đơn giản.

MS

MessagesState

Convenient state schema for chat agents
Khái niệm Prebuilt state có key messages với reducer add_messages.
Khi dùng Chatbot, tool-calling agent, conversational workflow.
Import from langgraph.graph import MessagesState
Extend Subclass thêm keys như user_id, summary.
from langgraph.graph import MessagesState

class AgentState(MessagesState):
    user_id: str
    summary: str

def call_model(state: AgentState):
    response = model.invoke(state["messages"])
    return {"messages": [response]}

graph.invoke({
    "messages": [{"role": "user", "content": "Hi"}],
    "user_id": "u-42",
})

Best: dùng MessagesState cho chat graph để tránh tự viết reducer messages.

Mistake: overwrite toàn bộ messages khi muốn append response mới.

TN

ToolNode

Prebuilt node for executing tool calls
Khái niệm Node chạy tool calls từ AIMessage và trả ToolMessages vào state.
Khi dùng ReAct/tool-calling agents thay vì tự parse tool calls.
Import from langgraph.prebuilt import ToolNode, tools_condition
Methods Dùng như node trong graph: builder.add_node("tools", ToolNode(tools))
from langchain.tools import tool
from langgraph.prebuilt import ToolNode, tools_condition

@tool
def get_weather(city: str) -> str:
    """Get current weather."""
    return f"{city}: 28C"

tools = [get_weather]
model_with_tools = model.bind_tools(tools)

def chatbot(state: MessagesState):
    return {"messages": [model_with_tools.invoke(state["messages"])]}

builder.add_node("chatbot", chatbot)
builder.add_node("tools", ToolNode(tools))
builder.add_conditional_edges("chatbot", tools_condition)
builder.add_edge("tools", "chatbot")

Best: dùng tool schema rõ và kiểm soát side effects bằng interrupt khi cần.

Mistake: tự thực thi tool calls thủ công khi ToolNode đã xử lý chuẩn.

CP

Persistence & Checkpointers

Threads, checkpoints, resume, time travel
Khái niệm Checkpointer lưu state snapshots theo thread để resume hoặc inspect history.
Khi dùng Memory, human approval, fault tolerance, long-running agents.
Import InMemorySaver, Postgres/SQLite savers cho production.
Config {"configurable": {"thread_id": "..."}}
from langgraph.checkpoint.memory import InMemorySaver

checkpointer = InMemorySaver()
graph = builder.compile(checkpointer=checkpointer)

config = {
    "configurable": {"thread_id": "conversation-42"}
}

graph.invoke(
    {"messages": [{"role": "user", "content": "Remember me"}]},
    config,
)

state = graph.get_state(config)
history = list(graph.get_state_history(config))

# Production: use durable checkpoint store,
# e.g. PostgresSaver or SqliteSaver.

Best: thread_id ổn định theo conversation; dùng durable saver trong production.

Mistake: dùng InMemorySaver cho dữ liệu thật hoặc không quản lý checkpoint growth.

STO

Stores & Long-Term Memory

Cross-thread persistence for user facts
Khái niệm Store lưu dữ liệu ngoài thread, truy cập qua namespace/key.
Khi dùng User profile, preferences, facts cần nhớ xuyên conversations.
Import from langgraph.store.memory import InMemoryStore
Methods put, get, search, delete
from langgraph.store.memory import InMemoryStore
from langgraph.runtime import Runtime

store = InMemoryStore()
graph = builder.compile(store=store)

def save_preference(state, runtime: Runtime):
    user_id = runtime.context["user_id"]
    runtime.store.put(
        ("users", user_id),
        "preference",
        {"language": "vi"},
    )
    return {}

config = {"configurable": {"thread_id": "t-1"}}
context = {"user_id": "u-42"}
graph.invoke(input_state, config=config, context=context)

Best: tách short-term thread state và long-term user memory rõ ràng.

Mistake: lưu PII không kiểm soát hoặc dùng store thay cho checkpoint state.

INT

Interrupts

Pause graph for human input or approval
Khái niệm interrupt() pause execution, resume bằng Command(resume=...).
Khi dùng Approval trước tool nguy hiểm, ask user, edit state, review output.
Import from langgraph.types import interrupt, Command
Yêu cầu Cần checkpointer để resume graph.
from langgraph.types import interrupt, Command

def approve_payment(state: State):
    answer = interrupt({
        "question": "Approve payment?",
        "amount": state["amount"],
    })
    if answer["approved"]:
        return {"approved": True}
    return {"approved": False}

graph = builder.compile(checkpointer=checkpointer)
config = {"configurable": {"thread_id": "pay-1"}}

graph.invoke({"amount": 100}, config)
graph.invoke(Command(resume={"approved": True}), config)

Best: interrupt trước side effects, kèm payload đủ để UI hiển thị quyết định.

Mistake: đặt interrupt trong node không idempotent rồi resume gây chạy lại side effect.

STR

Streaming

Values, updates, messages, events, custom data
Khái niệm Stream state updates, full values, LLM tokens, debug data hoặc custom events.
Khi dùng Chat UI, progress display, tool-call visibility, observability.
Modes values, updates, messages, debug, custom
Methods stream, astream, astream_events
for chunk in graph.stream(
    {"messages": [{"role": "user", "content": "Hi"}]},
    stream_mode="updates",
):
    print(chunk)

for token, metadata in graph.stream(
    input_state,
    stream_mode="messages",
):
    print(token.content, end="")

async for event in graph.astream_events(input_state):
    print(event["event"], event.get("name"))

Best: dùng updates cho progress, messages cho token UI, events cho tracing.

Mistake: giả định chunk format giống nhau giữa các stream modes.

SUB

Subgraphs

Compose graphs inside graphs
Khái niệm Một compiled graph có thể dùng như node trong graph khác.
Khi dùng Multi-agent, reusable workflows, team/subtask isolation.
Shared State Parent và subgraph có thể share key hoặc transform IO.
Methods Compile subgraph rồi add_node("name", subgraph).
class SubState(TypedDict):
    question: str
    answer: str

sub = StateGraph(SubState)
sub.add_node("answer", lambda s: {"answer": "42"})
sub.add_edge(START, "answer")
sub.add_edge("answer", END)
subgraph = sub.compile()

class ParentState(TypedDict):
    question: str
    answer: str
    final: str

parent = StateGraph(ParentState)
parent.add_node("qa", subgraph)
parent.add_node("finalize", lambda s: {"final": s["answer"]})
parent.add_edge(START, "qa")
parent.add_edge("qa", "finalize")
parent.add_edge("finalize", END)

Best: dùng subgraph để đóng gói workflow có ownership/schema rõ.

Mistake: share quá nhiều state keys khiến parent/subgraph coupling cao.

FN

Functional API

Workflow and task decorators
Khái niệm Viết workflow bằng decorators thay vì khai báo graph thủ công.
Khi dùng Flow giống code thường nhưng vẫn cần persistence/streaming.
Import from langgraph.func import entrypoint, task
Methods Task gọi bằng .result(); workflow invoke/stream như graph.
from langgraph.func import entrypoint, task
from langgraph.checkpoint.memory import InMemorySaver

@task
def research(topic: str) -> str:
    return f"facts about {topic}"

@task
def write_answer(facts: str) -> str:
    return f"Answer: {facts}"

@entrypoint(checkpointer=InMemorySaver())
def workflow(topic: str) -> str:
    facts = research(topic).result()
    return write_answer(facts).result()

workflow.invoke("LangGraph")

Best: dùng Functional API cho workflow tuyến tính/imperative dễ đọc.

Mistake: dùng khi cần graph topology phức tạp, nhiều route quan sát trực quan.

AG

Prebuilt Agents

create_react_agent and production-ready agent loops
Khái niệm Prebuilt agent graph cho model + tools + state + checkpointer.
Khi dùng Muốn ReAct/tool-calling nhanh mà vẫn có LangGraph state/persistence.
Import from langgraph.prebuilt import create_react_agent
Methods invoke, stream, get_state nếu compile với checkpointer.
from langgraph.prebuilt import create_react_agent
from langchain.chat_models import init_chat_model
from langchain.tools import tool

@tool
def search(query: str) -> str:
    """Search trusted docs."""
    return "LangGraph supports durable state."

model = init_chat_model("openai:gpt-5.5")
agent = create_react_agent(
    model=model,
    tools=[search],
    prompt="You are a concise assistant.",
    checkpointer=checkpointer,
)

result = agent.invoke(
    {"messages": [{"role": "user", "content": "What is LangGraph?"}]},
    {"configurable": {"thread_id": "agent-1"}},
)

Best: bắt đầu với prebuilt agent, rồi custom StateGraph khi cần control flow riêng.

Mistake: tự dựng ReAct loop từ đầu khi prebuilt agent đã đáp ứng.

MEM

Memory Patterns

Short-term, long-term, summarization
Short-term Checkpointer + thread state giữ conversation history.
Long-term Store lưu facts/preferences xuyên threads.
Context Runtime context truyền user_id, tenant_id, request scope.
Manage Trim/summarize messages để tránh vượt context.
from langgraph.graph import MessagesState

class ChatState(MessagesState):
    summary: str

def summarize_if_long(state: ChatState):
    messages = state["messages"]
    if len(messages) < 12:
        return {}
    summary = model.invoke([
        {"role": "system", "content": "Summarize chat."},
        *messages,
    ])
    return {
        "summary": summary.content,
        "messages": messages[-6:],
    }

Best: phân biệt memory phục vụ context và memory lưu tri thức user.

Mistake: đưa toàn bộ history vào prompt vô hạn hoặc lưu memory chưa được xác nhận.

TT

Time Travel & State Editing

Inspect, fork, replay, update graph state
Khái niệm Dùng checkpoints để xem history, resume từ checkpoint, fork execution.
Khi dùng Debug agent, fix human input, replay workflow, audit path.
Methods get_state, get_state_history, update_state
Yêu cầu Graph compile với checkpointer.
config = {"configurable": {"thread_id": "debug-1"}}

graph.invoke(input_state, config)

current = graph.get_state(config)
for checkpoint in graph.get_state_history(config):
    print(checkpoint.next, checkpoint.values)

graph.update_state(
    config,
    {"approved": True},
    as_node="human_review",
)

graph.invoke(None, config)

Best: dùng state history để debug routing và tool outputs thay vì chỉ print.

Mistake: update state sai node/context làm replay khó hiểu.

MAP

Quick Decision Map

Pick the right LangGraph component
  • StateGraph
    Workflow nhiều node, shared state, routing rõ.
  • MessagesState
    Chat/tool-calling graph có message history.
  • Reducer
    Nhiều node/branch update cùng key hoặc cần append.
  • Conditional edges
    Routing dựa trên state sau node.
  • Command
    Node vừa update state vừa chọn node kế tiếp.
  • Send
    Fan-out động, map-reduce, parallel per item.
  • Checkpointer
    Resume, memory theo thread, interrupts, time travel.
  • Store
    Long-term memory xuyên nhiều threads.
  • Interrupt
    Human approval/input trước bước rủi ro.
  • ToolNode
    Execute tool calls chuẩn trong agent loop.
  • Functional API
    Workflow imperative, tuyến tính, ít topology phức tạp.
  • Subgraph
    Đóng gói workflow/team/agent con có schema riêng.
